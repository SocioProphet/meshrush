# Network Ops Adapters

## 1. Scope

This specification defines adapter expectations for projecting MeshRush network-intelligence evidence into adjacent SocioProphet planes and operational repositories.

MeshRush remains the graph-operating runtime. It does not become the workspace controller, governance plane, learning plane, service mesh, or global ops intelligence platform.

## 2. Adapter doctrine

Adapters MUST preserve:

- graph-view identity
- evidence package references
- metric assumptions
- policy scope
- replayability references
- artifact lifecycle state
- warnings and invalidity conditions

Adapters MUST NOT silently reinterpret metric semantics.

## 3. Sociosphere-facing adapter

Expected contract:

- workspace context input
- session/run identifiers
- graph-view selection
- artifact persistence hooks
- state handoff points
- policy/context boundary references

## 4. agentplane-facing adapter

Expected contract:

- run manifests
- replayable traces
- evidence packages
- rollout metadata
- versioned artifact references
- stop/continue decision receipts
- failure and defer reasons

## 5. Alexandrian Academy-facing adapter

Expected contract:

- trajectories and episodes
- compiled artifact records
- experiment outputs
- probe/evaluation outputs
- policy-context references
- failure cases for review and curriculum

## 6. global-devsecops-intelligence adapter

Expected contract:

- topology-risk findings
- low-conductance cuts
- bottleneck nodes and edges
- blast-radius and propagation-risk features
- partial-visibility warnings
- service/dependency graph evidence
- incident or alert relevance expansion traces

## 7. memory-mesh adapter

Expected contract:

- reusable diffusion traces
- Katz relevance snapshots
- crystallized local subgraph artifacts
- conductance/cut artifacts
- exp-min estimator receipts
- gossip-min trace records

## 8. cairnpath-mesh adapter

Expected contract:

- path-quality metrics
- bottleneck edge/node references
- route-risk evidence
- cut-aware traversal constraints
- path reuse and path invalidation hints

## 9. orion-field-intelligence adapter

Expected contract:

- field topology summaries
- operational situational-awareness features
- propagation and bottleneck observations
- source-region trapped warnings
- frontier crossing events

## 10. agent-registry adapter

Expected contract:

Agents that provide network-intelligence capabilities SHOULD declare capability metadata for:

- random-walk analysis
- stationary distribution estimation
- Katz relevance scoring
- spectral-gap estimation
- conductance estimation
- gossip simulation
- Cheeger consistency checking
- exp-min distributed aggregation

## 11. socioprophet-agent-standards adapter

Expected contract:

The agent standards surface SHOULD define canonical vocabulary for:

- graph view
- transition model
- diffusion state
- stop/continue/defer decision
- conductance evidence
- spectral-gap evidence
- gossip trace
- Cheeger consistency
- exp-min estimator
- gossip-min trace

## 12. Service-mesh pattern adapters

Repositories such as Kiali and Kuma are relevant as design-pattern sources for topology visibility, policy surfacing, and service-mesh control-plane interoperability.

MeshRush SHOULD borrow useful observability and policy interaction patterns without inheriting unnecessary platform bloat.

## 13. Non-goals

This spec does not implement any adapter. It defines the contract surface that future adapter specs and implementation work should satisfy.
